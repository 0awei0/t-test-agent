# AI 测试官比赛优化计划

## 1. 目标与截止时间

- 比赛方向：方向二「AI 测试官」。
- 作品提交截止：2026-07-19 23:59:59。
- 内部完成目标：2026-07-18 前完成提交，7 月 19 日只保留阻断问题缓冲。
- 最终目标：形成一个可从干净 checkout 复现、可稳定现场演示、可通过 EdgeOne Makers 分享，并能让评委直接判断 Agent 价值的版本。

## 2. 比赛主线

- 主展示案例：`release-guard`。
  - 展示变更理解、风险识别、单测、API、浏览器验证、截图证据和发布阻断结论。
- 快速兜底案例：`agent-loop`。
  - 在 30 秒左右展示 Agent 多轮工具调用和决策过程。
- 修复对照案例：`release-guard-pass`。
  - 证明相同发布门禁能够识别修复完成并给出通过结论。
- 真实 MR：只作为评委追问时的增强材料，不作为现场主演示依赖。
- 最终比赛入口：EdgeOne Makers。
- FUE：保留为内部分享或备用入口。

## 3. 范围边界

### 本轮必须完成

- Agents SDK 主实现完成 Git 交付收口。
- 主展示和修复对照案例稳定运行。
- 建立一键比赛发布门禁。
- 报告突出 Agent 的自主理解、策略取舍和证据链。
- EdgeOne 页面能够回放一次真实执行结果。
- 完成干净环境验证、部署和现场彩排。

### 截止前暂不扩展

- 完整 TAPD 工作台。
- 定时线上巡检平台。
- 企业微信入站对话机器人。
- 自动修改、提交或发布真实业务仓库。
- 多租户任务管理平台。
- 任意仓库的云端代码执行服务。

#### 验收标准

```bash

uv lock --check
uv run ruff check .
uv run python -m unittest discover -s agents-sdk-agent/tests -p 'test_*.py' -v
npm --prefix frontend run typecheck
npm --prefix frontend run build
git status --short
```

要求：干净 checkout 可运行；验证全部通过；Git 中不存在敏感或生成数据；最终工作区干净。

#### 目标证据链

```text
读取 diff
  → 判断业务意图和风险
  → 选择单测/API/浏览器验证
  → 执行测试并发现失败
  → 读取失败日志
  → 生成或补充边界测试
  → 再次验证
  → 输出发布结论和建议动作
```
