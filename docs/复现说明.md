# AI 测试官复现说明

本文用于复现比赛展示中的模拟 TAPD / 工蜂 MR 选择、合成测试执行和报告导出流程。所有案例均为合成数据，**不会请求真实 TAPD、工蜂或业务仓库**。

## 1. 环境准备

在仓库根目录执行：

```bash
cd /path/to/t-test-agent
uv sync --locked --extra e2e --group dev
uv run python -m playwright install chromium
```

要体验模型驱动的 `agent-strict` 流程，在本地 `.env` 配置模型凭据（不要提交）：

```bash
ARK_API_KEY=<your-key>
AI_TEST_OFFICER_MODEL=doubao-seed-2-1-turbo-260628
AI_TEST_OFFICER_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
AI_TEST_OFFICER_OPENAI_API=chat_completions
AI_TEST_OFFICER_DISABLE_TRACING=true
```

## 2. 网页模拟流程

1. 在 EdgeOne Makers 控制台打开该项目的最新「预览」链接。
2. 报告首页点击 **体验模拟 TAPD / 工蜂测试任务**。
3. 从「模拟 TAPD 需求」下拉框选择需求背景。
4. 在 15 条「模拟工蜂 MR」案例中翻页，每页 5 条；绿色标签为**预期通过**，红色标签为**预期阻断**。
5. 选择一条 MR 后，点击 **模拟生成测试计划**。
6. 页面会明确显示本次验证范围，并可复制对应的本地执行命令；网页本身只模拟交互，**不会**在 EdgeOne 静态站点执行测试、访问真实 TAPD / 工蜂，或保存任何凭据。

## 3. 本地实际运行

例如复现优惠 / 库存 / 幂等风险：

```bash
uv run ai-test-officer demo run \
  --scenario promotion-chain \
  --demo-root runs/demos \
  --runs-root runs/reproduce \
  --run-id promotion-chain-reproduce \
  --planner-mode agent-strict \
  --allow-temp-test-code \
  --visualize \
  --dashboard-port 8789 \
  --env .env
```

命令会在 `runs/reproduce/promotion-chain-reproduce/` 创建隔离工作区、生成临时回归测试、执行验证并输出：

- `report.md`：测试结论与根因说明。
- `report.html`：可本地打开的报告。
- `run.json`：本地完整运行记录，仅供调试，不对外分享。
- `repo/tests/`：Agent 生成的临时测试，仅存在隔离工作区。

执行时会启动实时仪表盘；在浏览器打开 `http://127.0.0.1:8789/?run_id=promotion-chain-reproduce`，可看到真实命令、失败日志与最终结论。运行结束后按 `Ctrl+C` 退出本地仪表盘。

未配置模型时，可把 `--planner-mode agent-strict` 替换为 `--planner-mode deterministic`，用于离线检查报告与导出结构；该模式不展示真实 Agent 工具调用。

### 推荐默认展示案例

比赛演示建议先运行 `release-guard`：该案例会真实执行优惠边界单测、下单 API 和 Playwright 浏览器验证，并保存 `unsafe-coupon.png` 截图。它适合先向评委展示“阻断原因 + 命令 + 页面证据”的完整闭环：

```bash
uv run ai-test-officer demo run \
  --scenario release-guard \
  --demo-root runs/live-demos \
  --runs-root runs/live-runs \
  --run-id release-guard-demo \
  --planner-mode agent-strict \
  --allow-temp-test-code \
  --visualize \
  --dashboard-port 8789 \
  --env .env
```

## 5. 导出与安全检查

运行结束后，如需导出公开展示包：

```bash
npm --prefix frontend run build
uv run ai-test-officer report export-fue \
  --report runs/reproduce/promotion-chain-reproduce/report.md \
  --output runs/edgeone-site/promotion-chain-reproduce \
  --project-slug ai-test-officer-promotion-chain
uv run ai-test-officer demo doctor \
  --fue-public runs/edgeone-site/promotion-chain-reproduce/public \
  --env .env
```

通过 `demo doctor` 后，再按 [`部署.md`](部署.md) 部署到 EdgeOne Makers。导出包只包含脱敏后的报告、公开运行摘要和必要证据；完整 `run.json`、本地日志和密钥不得上传或提交。

CLI 部署必须从 `public/` 目录执行，避免线上根路径落成 404：

```bash
cd runs/edgeone-site/promotion-chain-reproduce/public
PAGES_SOURCE=skills edgeone makers deploy \
  -n ai-test-officer-promotion-chain \
  --json
```

如模型凭据、Playwright 或浏览器不可用，先使用 `--planner-mode deterministic` 验证本地导出结构；不要为了完成演示而关闭安全检查、在静态站中写入凭据，或把真实内部链接和日志直接上传。
