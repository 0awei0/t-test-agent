# AI 测试官比赛展示说明

## 演示数据边界

比赛入口中的 TAPD 链接、工蜂 MR 链接和下拉选项均为**模拟数据**，用于让评委快速体验“输入测试任务 → 理解变更 → 规划验证 → 输出发布建议”的交互流程。

- 页面不会请求模拟链接，不会读取真实 TAPD、工蜂或业务仓库数据。
- 演示场景只在本地隔离工作区生成合成订单、优惠、库存和支付重试数据。
- 导出的页面只包含脱敏后的结论、事件摘要和合成截图；不会包含 token、真实业务数据、完整日志或本地绝对路径。

## 本地实际运行合成测试

网页中的下拉框只用于模拟输入，不会在 EdgeOne 静态站点执行测试。需要实际运行时，在仓库根目录执行：

```bash
scripts/showcase_release_guard.sh
```

`release-guard` 是比赛主演示，覆盖优惠、库存、支付幂等、API 和浏览器证据；
`agent-loop` 是 30 秒快速兜底。两者都会在隔离工作区创建合成案例、执行测试并生成报告，
不访问真实 TAPD、工蜂或业务仓库。最终公开包通过 EdgeOne Makers 提交，FUE 仅作内部分享或备用入口。

## 后续真实接入

后续可在获得授权后，通过 `mcporter-internal` 的只读 TAPD 与工蜂能力接入真实上下文：读取需求、MR 元数据和 diff，再复用现有隔离工作区、命令白名单、临时测试路径限制和公开导出脱敏流程。

真实接入默认不执行提交、推送、合并、部署或其他远程写操作；涉及真实数据和截图时，需先完成数据最小化与脱敏审查。
